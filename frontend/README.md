# Payments Platform Navigator Frontend

React TypeScript frontend for the Payments Platform Navigator MVP.

The frontend consumes the deterministic FastAPI backend and presents role-based onboarding, payment flow exploration, service dependency context, rule-based Ask the Platform, change-safety checklist generation, knowledge-health metrics, and glossary search.

## Local Setup

```bash
cd frontend
npm install
```

## Backend Dependency

Start the backend first:

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

During local Vite development, the frontend defaults to the backend API at:

```text
http://localhost:8080
```

For production builds, if `VITE_API_BASE_URL` is not set, the frontend uses relative API paths such as `/api/services`. This is the expected mode for the single-container Cloud Run deployment where FastAPI serves both the frontend and backend from the same origin.

Override the API base URL explicitly when needed:

```bash
VITE_API_BASE_URL=http://localhost:8080 npm run dev
```

For a production build that targets a separate API origin:

```bash
VITE_API_BASE_URL=https://example-api-service npm run build
```

## Run

```bash
cd frontend
npm run dev
```

## Build

```bash
cd frontend
npm run build
```

## Ask the Platform: Deterministic vs AI-assisted Modes

The Ask the Platform feature supports two answer modes:

### Deterministic Mode (Default)
Rule-based answers generated directly from structured synthetic data (services, flows, runbooks, incidents, changes, tests, glossary). No external AI provider is called. The answer includes matched entities, relevant services/flows, risks, and source files with high transparency and explainability.

### AI-assisted Mode (Optional)
Grounded AI explanations generated by a configured provider (e.g., Vertex AI Gemini). The AI explanation is generated only from the deterministic context pack—it cannot hallucinate beyond the synthetic knowledge base. AI-assisted mode is disabled by default and requires backend configuration.

When AI is enabled:
- Backend must have `ENABLE_AI_EXPLANATIONS=true` and `AI_PROVIDER=vertex-gemini` (or equivalent).
- Frontend displays both the AI explanation (with confidence score) and deterministic answer in separate sections.
- Safety disclaimer notes that all answers are grounded in synthetic data only.
- If the AI provider fails or is unavailable, the system gracefully falls back to deterministic answers with a warning message.

When AI is disabled (default):
- Only the deterministic answer is displayed.
- The mode selector shows "Deterministic" as the only available option.
- All functionality remains unchanged and fully operational.

## Notes

- No authentication is implemented in the MVP.
- No database persistence is used.
- All visible data comes from the synthetic backend API.
- AI explanations are disabled by default; existing deterministic behavior is fully preserved.
- Routing uses browser hash state to keep the first frontend implementation simple.
